# Disputed chronology contract

A chronology is a reviewed view over candidate events and their source-linked assertions. It is not a declaration of historical truth.

Candidate events preserve all member assertions, independent-source keys, date ranges, participants, and event status. Affirmed and denied accounts remain visible in separate party lanes. Contested, ambiguous, inferred, and unresolved events generate review tasks rather than being silently smoothed into prose.

Corrections create immutable revisions, mark dependent artifacts stale, and queue targeted regeneration. Previously reviewed chronology snapshots remain available for audit.
