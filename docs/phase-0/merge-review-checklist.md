# Merge Review Checklist

Reviewers should confirm:

- [ ] The initial wedge is narrow and clearly labeled as a hypothesis.
- [ ] The repository does not claim customer validation is complete.
- [ ] Matter evidence remains distinct from legal authority.
- [ ] Allegations are not represented as facts.
- [ ] Source location and claim support are separate validation steps.
- [ ] Cross-matter leakage, hidden processing failure, and invalid citations remain zero-tolerance risks.
- [ ] Customer-content training is disabled by default as a policy baseline.
- [ ] Attorney approval is explicit and version-specific.
- [ ] Phase 1 is secure ingestion and provenance, not premature drafting.
- [ ] Proceed, pivot, pause, and kill gates are measurable.
- [ ] Research templates avoid storing confidential matter information publicly.
- [ ] Open assumptions and external evidence requirements remain visible.

Merge means adoption of the operating baseline. It does not authorize confidential production processing or waive any open external gate.
